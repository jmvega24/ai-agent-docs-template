# Orchestrator Checklist

Minimum checklist for the orchestration layer before closing work or moving focus.

## Per-turn checklist

- `Context read`: `AGENTS.md`, `CONTEXT.md`, architecture, relevant ADRs, plans, logs, and affected code.
- `Delimitation stated`: scope, exclusions, risks, assumptions, and target outcome.
- `Current role declared`: analysis, handoff, audit, sync, or local documentation adjustment.
- `Plans reviewed`: active work tracking is aligned with actual state.
- `Living docs reviewed`: context, contracts, logs, and architecture notes are still consistent.
- `Closure status chosen`: reported status matches actual evidence.

## Allowed status outputs

- `implementation in progress`
- `implementation ready`
- `pending audit`
- `pending sync`
- `closed`
