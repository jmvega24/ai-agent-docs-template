# ADR Guide

## Purpose

Architecture Decision Records document important technical decisions.

## When to write an ADR

Create an ADR when a decision:

- changes architecture,
- affects long-term maintainability,
- introduces a new pattern,
- changes operational behavior,
- or has meaningful tradeoffs.

## Naming

Use numeric prefixes:

- `0001-architecture-choice.md`
- `0002-auth-strategy.md`
- `0003-data-model.md`

## Minimum structure

- Context
- Decision
- Alternatives considered
- Consequences
